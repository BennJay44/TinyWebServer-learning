# 第 01 课 项目结构总览与学习路线（详细教程版）

## 0. 这节课你要达到什么程度
这不是一份“听完就结束”的讲义，而是一份“按步骤完成就能掌握”的教程。你学完本课后，应该能做到三件事：
- 不看代码细节，也能说清项目的整体结构。
- 能从入口函数讲到一次请求处理完成的大致路径。
- 能为自己制定一个可执行的复现计划，不会学到一半迷路。

建议学习时长：120 到 150 分钟。

## 1. 学习前准备

### 1.1 你需要的基础
- C++：类、构造析构、模板、STL、指针。
- Linux：进程/线程、文件描述符、网络基础概念。
- 工具：可以查看目录与源码的编辑器。

### 1.2 本课学习方法
按“先全局、后局部”执行：
1. 先看结构，不钻函数细节。
2. 再画图，把脑内认知外化。
3. 最后用自己的语言复述。

如果你觉得自己“好像懂了”，但复述不出来，说明还没有真正掌握。

## 2. 第一步：先建立项目全局地图

### 2.1 目录分层理解
请先把项目目录按职责分层，不要按字母顺序看。

- 启动与装配层：`main.cpp`、`config.cpp`、`webserver.cpp`
- 协议处理层：`http/http_conn.cpp`
- 并发处理层：`threadpool/threadpool.h`
- 连接管理层：`timer/lst_timer.cpp`
- 资源支持层：`log/`、`CGImysql/`、`lock/`
- 页面资源层：`root/`
- 压测层：`test_pressure/`

### 2.2 你要立刻回答的 3 个问题
1. 入口在哪？
2. 核心事件循环在哪？
3. HTTP 解析在哪？

参考答案：
- 入口在 `main.cpp`
- 核心事件循环在 `webserver.cpp`
- HTTP 解析在 `http/http_conn.cpp`

如果你回答不出来，请回到 2.1 再看一遍目录分层。

## 3. 第二步：抓住主流程，不陷入细节

### 3.1 从入口出发的启动流程
先看 `main.cpp`，关注函数调用顺序，不要逐行分析。

你要观察到这样的启动链路：
1. 解析配置参数。
2. 初始化 WebServer 对象。
3. 初始化日志、数据库连接池、线程池。
4. 初始化触发模式与监听。
5. 进入事件循环。

这一段的核心目标是：你要知道“系统先把地基搭好，最后再进入循环处理请求”。

### 3.2 事件循环是系统心脏
再看 `webserver.cpp`，找到 `eventLoop`。

你要理解的是：
- `epoll_wait` 等待事件。
- 根据事件类型分发：新连接、读事件、写事件、异常、信号。
- 每种事件走不同处理函数。

你暂时不用记住所有变量，只要记住一件事：
“这个项目本质是一个事件分发器 + 多个处理模块”。

## 4. 第三步：理解一次请求如何走完整链路

请你把以下路径写在纸上或笔记里，并在每个节点旁边标注文件名：

1. 客户端连接到服务端（监听 socket 收到连接）
2. accept 新连接并加入 epoll
3. 收到 EPOLLIN，读取请求数据
4. 解析 HTTP 报文
5. 执行业务逻辑（含数据库访问场景）
6. 组装 HTTP 响应
7. 收到 EPOLLOUT，发送响应
8. 根据 keep-alive 与超时决定复用或关闭连接

这是本课程后续 29 节课的主线。后面每一课都在补这 8 步中的某几个步骤细节。

## 5. 第四步：把两条学习主线分开

### 5.1 功能主线
目标：保证“正确”。
- 能登录注册
- 能请求静态资源
- 能正确返回 HTTP 响应

### 5.2 性能主线
目标：理解“快和稳”。
- LT 与 ET 的区别
- Reactor 与 Proactor 的区别
- 线程池调度
- 日志同步/异步策略
- 定时器回收非活跃连接

你必须同时学习两条主线。只学功能会变成“会跑 demo”；只学性能会变成“概念很懂但项目跑不稳”。

## 6. 本课核心练习（必须完成）

### 练习 1：模块关系图（20 分钟）
任务要求：
- 画出至少 8 个模块节点：入口、配置、事件循环、HTTP、线程池、定时器、日志、数据库连接池。
- 画出调用方向箭头。

最低标准：别人看你的图，能知道每个模块干什么。

### 练习 2：请求主路径图（25 分钟）
任务要求：
- 画出 8 个节点：accept、read、parse、process、build response、write、keep-alive、timeout close。
- 每个节点标注“主线程”还是“工作线程”。

最低标准：你自己能按图口述 3 分钟不中断。

### 练习 3：4 周学习计划（15 分钟）
模板如下：
- 第 1 周：跑通项目 + 读主流程
- 第 2 周：HTTP 状态机 + 线程池
- 第 3 周：日志/定时器/数据库连接池
- 第 4 周：压测 + 问题定位 + 小改造

要求：写明每周投入时长和验收产出。

## 7. 教学提问脚本（你可以自测）

请不看代码回答下面问题：
1. 为什么先看 `main.cpp` 再看 `http/http_conn.cpp`？
2. `eventLoop` 在系统里扮演什么角色？
3. 线程池在请求处理链路中大概插在哪一步？
4. 为什么定时器模块和连接管理耦合紧密？

参考要点：
- 顺序上必须先全局再局部，否则会迷失在函数细节。
- 事件循环负责统一接收并分发事件。
- 线程池承接部分读写后的业务处理任务。
- 超时连接的清理直接影响 fd 生命周期。

## 8. 常见误区与纠正方式

### 误区 1：从 `http_conn.cpp` 第一行硬啃
问题：会陷入细节，无法建立结构感。
纠正：先完成“目录分层图 + 主路径图”，再下钻模块。

### 误区 2：不做笔记，只看不画
问题：短期记忆很快丢失。
纠正：每节课至少产出 1 张图和 1 页结构笔记。

### 误区 3：把“能运行”当作“已掌握”
问题：面试或复现时讲不清设计取舍。
纠正：必须通过“口述 + 图示 + 实验记录”三重验收。
