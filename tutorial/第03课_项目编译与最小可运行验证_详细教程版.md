# 第 03 课 项目编译与最小可运行验证（详细教程版）

## 0. 本课目标
这节课要完成两件事：
- 你能独立完成 TinyWebServer 的编译，不依赖别人代操作。
- 你能完成“最小可运行验证”，确认程序确实启动并可被访问。

注意：本课是“构建与启动验证课”，不是“业务功能课”。登录注册、数据库细节会在后续课程展开。

建议时长：120 到 150 分钟。

## 1. 学习前检查（5 分钟）
你在进入本课前，至少满足以下条件：
- 已完成 WSL2 基础环境配置。
- g++、make、mysqlclient 已安装并可识别。
- 项目源码目录存在并可读写。

建议在终端先执行：

~~~bash
g++ --version
make --version
ldconfig -p | grep -i mysqlclient
~~~

若三条命令都正常输出，再进入下一步。

## 2. 先理解“编译到底在做什么”（10 分钟）
不要急着敲命令，先读构建入口：
- build.sh
- makefile

你需要搞清楚 3 件事：
1. 最终生成的可执行文件名是什么。
2. 编译用到了哪些源文件。
3. 需要链接哪些库。

你应当得到结论：
- 目标文件：server
- 链接库：pthread、mysqlclient

理解这一层后，你排错会快很多。

## 3. 第一步：确认工作目录（5 分钟）

在 WSL 终端执行：

~~~bash
pwd
ls
~~~

然后切换到项目根目录（示例）：

~~~bash
cd ~/work/TinyWebServer
~~~

再执行一次：

~~~bash
ls
~~~

你应该能看到：
- build.sh
- makefile
- main.cpp
- webserver.cpp
- 各子目录（http、log、timer、threadpool、CGImysql 等）

如果看不到，说明你目录不对，先不要继续。

## 4. 第二步：执行标准编译（15 分钟）

推荐方式 A（脚本入口）：

~~~bash
sh ./build.sh
~~~

等价方式 B（直接目标）：

~~~bash
make server
~~~

编译成功后，执行：

~~~bash
ls -l server
file server
~~~

你应该看到：
- server 文件存在
- 是 ELF 64-bit 可执行文件

## 5. 第三步：做“最小启动验证”（20 分钟）

在项目根目录执行：

~~~bash
./server
~~~

你要观察两类现象：
1. 程序是否立即退出。
2. 程序运行时是否占住终端（一般会保持前台运行）。

如果程序保持运行，说明最小启动通过。

另开一个终端，做端口检查（默认 9006）：

~~~bash
ss -lntp | grep 9006
~~~

若输出里有 LISTEN 记录，说明监听成功。

## 6. 第四步：最小访问验证（20 分钟）

### 6.1 在 WSL 内部本地测试
在第二个终端执行：

~~~bash
curl -I http://127.0.0.1:9006
~~~

你希望看到 HTTP 响应头（例如状态行、Connection 等信息）。

### 6.2 在 Windows 浏览器测试
在浏览器打开：
- http://127.0.0.1:9006

如果页面可打开，说明“编译 + 启动 + 可访问”闭环成功。

## 7. 第五步：带参数启动并验证（20 分钟）

本项目支持命令行参数，先做一个简单验证：

~~~bash
./server -p 9007 -l 1 -m 0 -o 1 -s 8 -t 8 -c 0 -a 0
~~~

这条命令表示：
- 端口 9007
- 异步日志
- LT+LT
- 优雅关闭开启
- SQL 连接池 8
- 线程池 8
- 日志打开
- Proactor 模型

再检查监听：

~~~bash
ss -lntp | grep 9007
~~~

如果监听端口变为 9007，说明参数透传生效。

## 8. 常见错误排查（重点）

### 8.1 报错：Permission denied: ./build.sh
原因：脚本无执行权限。
处理：

~~~bash
chmod +x build.sh
sh ./build.sh
~~~

### 8.2 报错：mysql/mysql.h: No such file or directory
原因：MySQL 开发头文件缺失。
处理：

~~~bash
sudo apt install -y libmysqlclient-dev
# 或
sudo apt install -y default-libmysqlclient-dev
~~~

### 8.3 报错：cannot find -lmysqlclient
原因：链接阶段找不到库。
处理：
1. 执行 `ldconfig -p | grep -i mysqlclient`。
2. 若无输出，重新安装开发库。
3. 若有输出仍失败，检查多架构或库路径配置。

### 8.4 报错：Address already in use
原因：端口被占用。
处理：

~~~bash
ss -lntp | grep 9006
# 找到占用进程后停止，或改端口启动
./server -p 9007
~~~

### 8.5 现象：程序启动后秒退
可能原因：
- 数据库初始化失败
- 参数错误
- 环境变量或路径问题

处理策略：
1. 先确认是否能无参数启动。
2. 再逐项加参数定位。
3. 记录第一条错误日志，不要只描述“跑不起来”。

## 9. 本课核心练习（必须完成）

### 练习 1：构建链路复述
要求：用你自己的话解释从 makefile 到 server 可执行文件的过程。
最低标准：能说出“编译 -> 链接 -> 生成可执行文件”。

### 练习 2：最小可运行验证截图
要求：提交 3 张截图。
- `ls -l server`
- `ss -lntp | grep 9006`（或你设置的端口）
- 浏览器访问首页

### 练习 3：参数生效验证
要求：用自定义端口启动，并验证端口确实变化。
最低标准：`./server -p xxxx` + `ss` 验证成功。

## 10. 验收标准（严格版）
满足以下全部条件才通过：
1. 你能在空终端中独立完成编译。
2. 你能解释构建脚本和 makefile 的角色。
3. 你能启动服务并验证监听端口。
4. 你能通过 curl 或浏览器完成最小访问验证。
5. 你能排查至少一个常见错误并写出处理过程。
